# loadout — choose skills before doing work

Allocating the right skills IS the first move. Don't start a non-trivial task
without this pass. Fast (~10s), one shortlist.

## Procedure

1. **Classify the task.** One line: domain + intent + irreversibility.

2. **Scan the arsenal.** Match the task against `references/arsenal.md`
   (domain → trigger → skills). Pull only skills that change the outcome.
   Edit `arsenal.md` to list YOUR installed skills — it ships as a template.

3. **Shortlist.** Name 1-5 skills + one-line why each. Cut the rest.
   Precision beats coverage. Never load 10 skills — each one is context cost.

4. **Confirm token-savers ON.** The lean set (see arsenal.md) stays active every
   task. Prefer the lean variant of every tool.

5. **State the loadout, then proceed.** "Loadout: X (why), Y (why). Savers: on."

## Anti-patterns
- Loading skills "just in case" — every loaded skill is context cost.
- Guessing instead of scanning — you won't recall a 100+ skill arsenal.
- Reading whole files when a scoped search/graph query would answer.
